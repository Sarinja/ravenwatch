🐦 RavenWatch — Changelog

v0.6.0 — Target Accuracy + Travel Filtering

  • Reworked "Most Likely Targets" logic for improved real-world accuracy
  • Added battle stat ratio validation using FFScouter bsEstimate vs player total stats
  • Targets now evaluated using stat ratio, FF value, freshness, and data source
  • Added High / Medium / Low confidence scoring for all targets
  • Detects and penalizes mismatches between FF values and estimated stats
  • Prevents low-stat targets from incorrectly appearing as strong fair fights
  • Added estimated stat percentage display relative to player
  • Added FFScouter source display (premium vs bss)
  • Added mismatch warning notes for unreliable FF values
  • Fixed critical issue where traveling players appeared as targets
  • Expanded travel detection to include:
    • Returning to Torn from ...
    • Traveling to ...
    • Flying to ...
    • Landing in ...
    • Country-based travel status detection
  • Centralized attackability logic into a shared function
  • Traveling players are now fully excluded from target lists and activity calculations
  • Improved scoring stability under stale or partial FFScouter data
  • Reduced reliance on raw FFScouter values without validation
  • Updated target label to reflect validation logic

v0.5.0 — Licensing + Hardening

  • Replaced client-trusted license system with server-backed validation
  • Added Flask licensing API hosted on production VPS
  • Implemented secure activation, status, and token refresh endpoints
  • Added device-based activation tracking
  • Added admin controls: revoke, unrevoke, and reset activations
  • Implemented mid-session license validation for live enforcement
  • Added offline grace mode for temporary network outages
  • Improved license state handling across UI (licensed, offline, revoked, error)
  • Removed embedded license hash allowlist from client
  • Reduced attack surface by moving license authority fully server-side
  • Disabled global Tauri exposure (withGlobalTauri = false)
  • Added Content Security Policy (CSP)
  • Cleaned up legacy license code and dead references
  • Improved overall application integrity and distribution control

v0.4.1 — Status + Target Control

  • Improved Dashboard clarity by separating chain state from player state
  • Added configurable "How many FF targets?" setting
  • Users can now control how many targets are shown (default 5)
  • Integrated target count setting directly into scanner logic
  • Maintained compact UI while increasing flexibility
  • Refined Dashboard usability for faster decision making

v0.4.0 — War Targeting + Window Behavior

  • Added Pre-War Targets so players can preview likely targets before a war begins
  • Added War Targets for live war target suggestions during active faction wars
  • Added Save the Chain mode when a live chain becomes critical
  • Added optional FFScouter API key support for fair-fight target enrichment
  • Added RavenWatch fallback scoring when FFScouter data is unavailable
  • Added direct Attack and Profile buttons for suggested targets
  • Added compact target card layout for faster reading under pressure
  • Added explanatory info card under War Targets
  • Added FFScouter API key field to Settings
  • Refined war target labels to match context: Pre-War Targets, War Targets, and Save the Chain
  • Fixed Always on Top behavior on macOS by enabling the required Tauri capability
  • Preserved existing window size and position persistence while fixing Always on Top
